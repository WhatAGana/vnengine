# 배치 화면 — 설계 명세 (작동 확인용 최소 버전)

> 목적: 07-B 배치 로직(RoomGraph / PlacementValidator / MonsterDef.Cost)을 **화면에서 조작 가능하게** 하는 첫 UI.
> 오프닝 스모크(OpeningSmokeBootstrap)와 동일 패턴 — 부트스트랩으로 UI를 코드 생성해 씬에 배선.
> **예쁨 목표 아님.** "방에 몹이 쌓이고 예산이 깎이는 게 눈에 보인다"까지가 목표.

> **⚠ 07-B 코드 정정(2026-07-10):** 초안은 "방마다 고정 슬롯 격자(· = 빈 슬롯)"를 가정했으나 **07-B엔 슬롯 개념이 없다.** `RoomNode`는 `Defenders` 리스트만 갖고, 제약은 슬롯 개수가 아니라 **코스트 예산제**(총예산 = 방수 × 3). 또 배치 상태는 `RunState`에 없다(불변 `RoomNode`뿐). → UI를 "몹-목록 + 예산 바"로 바꾸고, 이번 스모크는 RunState 편입 없이 UI 로컬 `PlacementPlan`으로 진행(옵션 B).

---

## 1. 화면 구성 (위-가운데-아래 3단)

```
 [상단 바]   던전레벨 L1 · 예산 4 / 9              ← 사용코스트 / 총예산(방수×3)
─────────────────────────────────────────────────
 [방 영역]   방1          방2          방3     |코어|
            ┌────────┐  ┌────────┐  ┌────────┐  표시만
            │임프  C1│  │        │  │서큐  C3│  ← 방에 배치된 몹이 세로로 쌓임
            │오크  C2│  │        │  │        │     (고정 슬롯 아님, 목록)
            └────────┘  └────────┘  └────────┘
─────────────────────────────────────────────────
 [보유 몹]  [임프 C1][고블린 C1][오크 C2][하이오크 C3][서큐버스 C3][데스나이트 C4][고위마족 C5]
```

- **상단 바**: 던전레벨 + **예산 바**(사용코스트 / 총예산). 총예산 = 방수 × `PlacementValidator.BudgetPerRoom`(=3). 슬롯 개수 제약 아님.
- **방 영역**: `RoomGraph`의 방들을 가로로 나열. 각 방 = 고정 슬롯 격자가 아니라 **그 방에 배치된 몹이 세로로 쌓이는 목록**. 빈 방은 목록이 비어 있음. 코어는 표시만(배치 불가).
- **보유 몹**: `MonsterCatalog.Default()`의 몹들(이름 + 코스트). 클릭해서 선택 → 방 클릭으로 배치.

## 2. 조작 (클릭 기반 — 드래그는 나중)

**배치**:
1. 하단 보유몹 칸 클릭 → 선택됨(하이라이트).
2. 방 헤더(“+배치”) 클릭 → 그 방 목록에 몹 추가 시도.
3. `PlacementValidator.Validate`로 검증: 예산 초과면 **배치 거부**(상단에 "예산 초과" 표시, 목록 불변), 성공하면 방 목록에 몹 추가 + 예산 사용량 증가.

**제거**:
- 방에 쌓인 몹 칸 클릭 → 그 몹을 목록에서 제거, 예산 사용량 감소.

**주인공 배치(생략)**: 이번 최소버전에선 안 함(`PlacementPlan.HasHero=false`). 코어앞 제약은 후속.

## 3. 로직 연결 (07-B 기존 것 호출, 새 로직 만들지 말 것)

- 방 구조: `RoomGraph.Linear` (07-B). 이번 스모크는 **고정 3방**(예산 9)로 시작. (실제 방 개수 상한은 `DungeonRoomRule.RoomsCap` — 건설 슬라이스에서.)
- 배치 검증: `PlacementValidator.Validate`(방수×3 예산, `MonsterDef.Cost` 합) — **검증만 재사용.** `PlacementBuilder.Apply`(그래프 실체화)는 이번 스코프 밖(웨이브 전달 시 사용).
- 배치 상태: **이번 스모크는 `RunState`에 편입하지 않는다.** UI 부트스트랩이 `PlacementPlan`(`MonsterPlacement` 리스트)을 **로컬로** 들고 편집, 매 조작마다 `Validate` 호출로 예산/초과 표시.
  - 확인된 사실: `RoomNode`는 불변(`Defenders` 리스트, 슬롯 개수 없음), `RunState`엔 배치 필드 없음, 증분 place/remove 전이 함수 없음(플랜 전체→`Apply`만 있음).
  - **후속 독립 슬라이스**: 배치 상태를 `RunState`에 편입(회차 리셋) + 세이브 직렬화 + 웨이브 전투로 전달.
- **UI는 로직의 상태를 그리고, 조작 시 `Validate`로 게이트한다. 로직 신규 금지.**

## 4. 부트스트랩 (오프닝 패턴 재사용)

- `PlacementSmokeBootstrap.cs`(`OpeningSmokeBootstrap` 패턴): 카메라 · Canvas · EventSystem · UI를 코드로 생성. 씬 `PlacementSmoke.unity` 열고 Play하면 배치 화면.
- 방 슬롯/몹 칸 = 유니티 UI `Button`(+`Image`). 몹 = 텍스트(이름 + 코스트)로 충분(스프라이트 나중).
- 예산/사용량/상태 = 상단 `TMP_Text`. 방 목록은 `VerticalLayoutGroup`으로 쌓기.

## 5. 검증 (눈으로)

Play 후 확인:
- 방들이 그려지고, 보유 몹 목록이 코스트와 함께 뜨는가
- 보유몹 클릭→방 클릭 시 그 방 목록에 몹이 들어가는가
- 예산 사용량이 정확히 증가하는가 (방수×3 총예산)
- 예산 초과 시 배치가 막히고 "예산 초과"가 뜨는가
- 배치된 몹 클릭 시 제거되고 예산이 돌아오는가

## 6. 이번에 안 하는 것 (명시)

- 스프라이트/아트 (몹 = 텍스트로 충분)
- 드래그앤드롭 (클릭으로)
- 전투 연출 (배치만, 웨이브 실행은 별도)
- 예쁜 레이아웃/애니메이션
- 주인공 배치
- **RunState 편입 · 세이브 · 웨이브 전달 (후속 독립 슬라이스, §3)**
→ "방에 몹 쌓이고 예산 깎이는 것"만 눈으로 확인되면 이 슬라이스 성공.
