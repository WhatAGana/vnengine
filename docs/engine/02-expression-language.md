# 표현식 언어 스펙

`.vns`의 `$` 대입, `if`/`elif`/`while` 조건, `menu` 선택지 조건, `random()` 인자에는 모두 **표현식**이 옵니다.
이 문서는 그 표현식 언어의 타입·연산자·우선순위·평가 규칙을 정의합니다.

- 파서: `Assets/Scripts/VNEngine/Core/Expressions/ExprParser.cs` (재귀 하강)
- 평가기: `Assets/Scripts/VNEngine/Core/Expressions/ExprEval.cs`
- 값: `Assets/Scripts/VNEngine/Core/Values/VnValue.cs`

---

## 1. 값 타입

표현식과 변수의 값은 두 종류뿐입니다.

| 타입 | 예 | 비고 |
|---|---|---|
| **정수 (Int)** | `0`, `42`, `-7` | 32비트 정수. 실수(float)는 **미지원** |
| **불 (Bool)** | `true`, `false` | 비교·논리 연산의 결과 |

- 문자열은 값 타입이 아닙니다. 대사 텍스트나 `character name:`처럼 **리터럴로만** 쓰이고 변수에 담을 수 없습니다.
- 정수 리터럴은 음수 토큰이 따로 없고, `-7`은 **단항 마이너스**(`-`)를 `7`에 적용한 것으로 파싱됩니다.

### 진리값(Truthy) 규칙
조건 자리(`if`, `while`, `and`/`or`/`not`, 선택지 `if`)에서 값의 참/거짓은 다음과 같습니다.

- **0이 아니면 참, 0이면 거짓.** 이는 Int·Bool 공통입니다(`Bool(true)`는 내부적으로 1).
- 따라서 `if 골드:` 는 "골드가 0이 아니면"이라는 뜻이 됩니다(정수를 조건에 직접 써도 동작).

---

## 2. 변수

- 식별자(예: `골드`, `호감도`, `met_yor`)는 변수 참조입니다. 유니코드 글자·숫자·밑줄(`_`)로 구성되며 숫자로 시작할 수 없습니다.
- **정의되지 않은 변수를 읽으면 정수 `0`** 입니다. 별도 선언 없이 `$ 골드 += 10`이 `10`이 되는 이유입니다.
- 조건 문맥에서 정의 안 된 변수는 `0` → **거짓**으로 작동합니다(예: `if 만난적:` 은 아직 설정 전이면 거짓).
- 대입은 `.vns`의 `$` 명령으로만 합니다([01번 문서 §9](01-vns-language-reference.md)).

---

## 3. 리터럴 · 내장 함수

| 표기 | 의미 |
|---|---|
| `123` | 정수 리터럴 |
| `true` / `false` | 불 리터럴 |
| `random(a, b)` | `a` 이상 `b` **이하**의 정수 난수(양 끝 포함). `a`,`b`는 정수 표현식 |

- `random`의 난수원은 `VNRunner`의 `randomSeed`로 시드된 결정론적 PRNG(xorshift32)입니다.
  같은 시드 + 같은 실행 경로 → **같은 난수열**. 그래서 테스트가 재현 가능하고 세이브가 정확히 복원됩니다.
- `random(a, b)`의 인자가 정수가 아니면 런타임 에러(`random(a, b) requires integer bounds`).

## 4. 연산자

### 산술 (정수 연산)
| 연산자 | 의미 | 비고 |
|---|---|---|
| `+` `-` `*` | 덧셈·뺄셈·곱셈 | |
| `/` | **정수 나눗셈** | `7 / 2 == 3` (버림). `0`으로 나누면 런타임 에러 |
| `%` | 나머지 | `0`으로 나누면 런타임 에러 |
| `-x` | 단항 마이너스 | 피연산자는 정수여야 함 |

- 산술 연산자는 **정수 피연산자만** 받습니다. 불에 산술을 적용하면 런타임 에러
  (`Operator '+' requires integer operands`).

### 비교
| 연산자 | 의미 |
|---|---|
| `>` `<` `>=` `<=` | 크기 비교 — **정수 피연산자만** |
| `==` `!=` | 같음/다름 — **타입 + 값**으로 비교 |

- `>`, `<`, `>=`, `<=`는 정수끼리만 가능합니다(불에 쓰면 런타임 에러).
- `==`/`!=`는 **종류(Kind)와 값이 모두 같아야 같음**입니다.
  → 교차 타입은 항상 다름: `1 == true` 는 **거짓**(Int 1 ≠ Bool true), `0 == false` 도 **거짓**.
  같은 타입끼리만 의미 있게 비교하세요.

### 논리 (단축 평가)
| 연산자 | 의미 |
|---|---|
| `and` | 좌변이 거짓이면 우변을 **평가하지 않고** 거짓. 결과는 불 |
| `or` | 좌변이 참이면 우변을 **평가하지 않고** 참. 결과는 불 |
| `not x` | x의 진리값을 뒤집은 불 |

- `and`/`or`는 **단축 평가**합니다. 예: `골드 >= 10 and 만난적` 에서 `골드 < 10`이면 `만난적`은 읽지 않습니다.
- 논리 연산의 결과는 항상 **Bool**입니다.

## 5. 연산자 우선순위

낮은 것 → 높은 것 순서(위가 가장 느슨하게 묶임):

```
or
and
==  !=  >=  <=  >  <      (비교)
+  -                       (덧셈/뺄셈)
*  /  %                    (곱셈/나눗셈/나머지)
-x   not x                 (단항)
( ) · 리터럴 · 변수 · random(...)   (일차식)
```

- 괄호 `( )`로 우선순위를 강제할 수 있습니다.
- 예: `호감도 >= 50 and not 거절함` → `(호감도 >= 50) and (not 거절함)`.
- 예: `(용기 + 지혜) * 2` → 괄호 없으면 `용기 + (지혜 * 2)`가 되므로 괄호 필수.

### 정식 문법 (BNF 요약)
```
or      := and ( "or" and )*
and     := cmp ( "and" cmp )*
cmp     := add ( ("=="|"!="|">="|"<="|">"|"<") add )*
add     := mul ( ("+"|"-") mul )*
mul     := unary ( ("*"|"/"|"%") unary )*
unary   := ("-"|"not") unary | primary
primary := INT | "true" | "false" | "random" "(" or "," or ")" | IDENT | "(" or ")"
```

## 6. 평가 규칙 요약

| 상황 | 결과 |
|---|---|
| 정의 안 된 변수 | 정수 `0` |
| `7 / 2` | `3` (정수 나눗셈, 버림) |
| `/ 0`, `% 0` | 런타임 에러 (Division/Modulo by zero) |
| `1 == true` | `false` (타입 다름) |
| `true + 1` | 런타임 에러 (산술은 정수만) |
| `골드 >= 10 and 미평가식` | `골드 < 10`이면 우변 미평가 |
| `random(1, 6)` | 1~6 정수(양 끝 포함), 시드 결정론적 |

---

## 7. 예시

```vns
$ 총점 = (용기 + 지혜) * 2
$ 주사위 = random(1, 6)
if 호감도 >= 50 and 만난적:
    ...
if not 거절함 or 골드 >= 100:
    ...
while 남은턴 > 0:
    $ 남은턴 -= 1
"주사위 결과: [주사위], 총점: [총점]"
```
