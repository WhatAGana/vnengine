# `.vns` 대본 언어 레퍼런스

`.vns`는 이 엔진의 전용 스토리 작성 언어입니다. Ren'Py 계열의 **들여쓰기 기반** 스크립트로,
대사·연출·선택지·조건 분기·변수·반복을 정수 인덱스나 JSON 없이 사람이 읽고 쓸 수 있게 표현합니다.

- 파일 확장자: `.vns`
- 인코딩: **UTF-8**
- 저장 위치: `Assets/Resources/<하위폴더>/` (기본 `scripts`) 아래. 폴더 안 모든 `.vns`가 한 프로그램으로 합쳐집니다.
- 이 문서에서 다루는 것: **문법 전체**. 표현식(변수·연산자)의 세부는 [02번 문서](02-expression-language.md).

---

## 1. 파일 구조의 기본 규칙

### 1.1 줄 · 들여쓰기

- **한 줄 = 한 명령.** 줄바꿈으로 명령을 구분합니다.
- **블록은 들여쓰기(스페이스)로 표현합니다.** `menu:`, `if:`, `while:`, 선택지 라벨 뒤의 본문은
  헤더보다 **더 깊게** 들여씁니다.
- 들여쓰기는 **스페이스만** 허용합니다. **탭을 쓰면 로드 시 에러**가 납니다
  (`tab used in indentation (use spaces)`).
- 들여쓰기 칸 수는 "부모보다 깊으면 자식"이라는 **상대 규칙**으로 판정합니다. 4칸을 권장하지만
  일관되기만 하면 2칸이든 4칸이든 됩니다. `elif`/`else`는 짝이 되는 `if`와 **정확히 같은 깊이**여야 합니다.

### 1.2 빈 줄 · 주석

- **빈 줄**은 무시됩니다(블록 구조에 영향 없음).
- **주석은 `#`** 로 시작해 줄 끝까지입니다.
  - 줄 전체 주석: 줄이 `#`로 시작.
  - 줄 끝 주석: `#` **앞에 스페이스가 있어야** 주석으로 인식합니다. 예) `show 요르 left   # 왼쪽 등장`
  - 큰따옴표 문자열 **안**의 `#`는 주석이 아니라 글자입니다. 예) `"해시태그 #1"` 은 그대로 출력됩니다.

```vns
# 이 줄 전체가 주석
요르 "안녕"        # 여기부터 주석 (앞에 공백 필요)
요르 "가격은 #1"   # 따옴표 안 #는 글자
```

## 2. 명령 요약표

| 명령 | 문법 | 의미 |
|---|---|---|
| 주석 | `# ...` | 무시 |
| 캐릭터 정의 | `character <id> name:"..." color:"#hex"` | 표시명·이름색 등록 (`color` 선택) |
| 라벨 | `label <이름>:` | 점프 대상 지점 |
| 대사 | `<id\|이름> "텍스트"` | 화자 있는 대사 |
| 나레이션 | `"텍스트"` | 화자 없는 대사 |
| 배경 | `bg <이름>` | 배경 설정 |
| 등장 | `show <캐릭터> <left\|center\|right>` | 슬롯에 캐릭터 표시 (위치 생략 시 center) |
| 퇴장 | `hide <캐릭터>` | 슬롯에서 제거 |
| 선택지 | `menu:` + `"라벨"[ if 조건]:` 블록들 | 플레이어 선택 분기 |
| 대입 | `$ <변수> = <식>` (및 `+= -= *= /=`) | 변수 변경 |
| 조건 | `if <식>:` / `elif <식>:` / `else:` | 조건 분기 |
| 반복 | `while <식>:` | 조건이 참인 동안 반복 |
| 점프 | `jump <라벨>` | 복귀 없는 이동 |
| 호출 | `call <라벨>` | 복귀하는 이동(서브루틴) |
| 복귀/종료 | `return` | 서브루틴 복귀 또는 스크립트 종료 |

각 명령의 상세는 아래.

## 3. 캐릭터 정의 — `character`

화자 id에 **표시명**과 **이름색**을 연결합니다. 보통 파일 상단에 둡니다.

```vns
character 요르 name:"요르 (숲의 요정)" color:"#8fd3ff"
character 나  name:"나"
```

- 문법: `character <id> name:"<표시명>" [color:"#RRGGBB"]`
- `name:` 은 **필수**, `color:` 는 선택입니다. 순서는 상관없지만 둘 다 `key:"값"` 꼴이어야 합니다.
- 대사에서 화자로 `id`를 쓰면 실행 시 **표시명과 색으로 치환**됩니다.
- **같은 `id`를 두 번 정의하면 컴파일 에러**(`duplicate character '<id>'`).
- 정의하지 않은 화자 이름을 대사에 그냥 써도 됩니다 — 그 경우 이름이 **글자 그대로** 표시되고 색은 없습니다(3.1의 "리터럴 화자").

## 4. 대사 · 나레이션

### 4.1 화자 있는 대사

```vns
요르 "주말에 뭐 할래?"
```

- 문법: `<화자> "텍스트"` — 화자는 큰따옴표 **앞의 텍스트**입니다.
- 화자가 `character`로 정의된 id면 → 표시명 + 색으로 렌더.
- 화자가 정의되지 않은 이름이면 → **그 이름 그대로** 표시(색 없음). 예) `점원 "어서 오세요"`.

### 4.2 나레이션 (화자 없음)

```vns
"…정적이 흘렀다."
```

- 줄이 큰따옴표로 시작하면 나레이션입니다(화자 = 없음).

### 4.3 텍스트 보간 `[변수]`

대사·나레이션 텍스트 안에서 `[변수이름]`은 **실행 시점의 값**으로 치환됩니다.

```vns
요르 "남은 골드는 [골드]개야."     # 골드가 30이면 → "남은 골드는 30개야."
"플래그 상태: [만난적]"            # bool이면 → "플래그 상태: true"
```

- `[[` 는 리터럴 대괄호 `[` 하나를 출력합니다.
- 정의 안 된 변수는 `0`으로 치환됩니다(표현식과 동일 규칙 → [02번 문서](02-expression-language.md)).
- 닫는 `]`가 없으면 그 지점부터 끝까지 글자 그대로 남습니다.

> 진행 방식: 대사 한 줄이 표시되면 엔진은 **플레이어 입력(클릭/탭)까지 대기**합니다. 입력이 오면 다음 명령으로 넘어갑니다.

## 5. 무대 연출 — `bg` · `show` · `hide`

```vns
bg city_sunset          # 배경 설정
show 요르 left           # 왼쪽 슬롯에 등장
show 민지 right          # 오른쪽 슬롯에 등장
hide 민지                # 민지 퇴장
```

| 명령 | 문법 | 비고 |
|---|---|---|
| `bg` | `bg <배경이름>` | 배경 상태를 설정. 실제 스프라이트 매핑은 Unity `StageViewUnity`에서 이름→이미지로 연결 |
| `show` | `show <캐릭터> [left\|center\|right]` | 위치 생략 시 `center`. 위치 이름이 셋 중 하나가 아니면 파싱 에러 |
| `hide` | `hide <캐릭터>` | 해당 캐릭터를 슬롯에서 제거 |

- `bg`/`show`/`hide`는 **즉시 실행**되고 대기하지 않습니다(대사와 달리 입력을 기다리지 않음).
- 무대 상태(배경·슬롯 점유)는 내부적으로 추적되어 **세이브/로드 시 그대로 복원**됩니다.

## 6. 선택지 (이벤트 분기의 핵심) — `menu`

플레이어에게 여러 선택지를 보여주고, 고른 것에 따라 분기합니다.

```vns
menu:
    "같이 영화 보자":
        $ 호감도 += 30
        jump 데이트
    "금화를 준다 (10 필요)" if 골드 >= 10:
        $ 골드 -= 10
        jump 뇌물
    "귀찮은데 집에":
        $ 호감도 -= 10
        jump 거절
```

### 문법
- 헤더는 정확히 `menu:` 한 줄이어야 합니다.
- 각 선택지는 헤더보다 깊게 들여쓴 **`"라벨"[ if <조건>]:`** 줄로 시작하고, 그 아래 더 깊게 들여쓴 본문 블록을 가집니다.
- 선택지 라벨은 반드시 **큰따옴표로 시작**하고 헤더 줄은 **`:`로 끝나야** 합니다.

### 조건부 선택지 — `if`
- `"라벨" if <조건>:` 형태로 쓰면, 실행 시 **조건이 참인 선택지만** 화면에 표시됩니다.
- 조건이 거짓이면 그 선택지는 **숨겨집니다**(회색 비활성이 아니라 목록에서 빠짐).
- 조건 없는 선택지는 항상 표시됩니다.

### 동작
- 플레이어가 하나 고르면 → 해당 본문 블록 실행 → 블록이 끝나면 `menu` **다음 줄로** 자동 진행합니다
  (본문에서 `jump`/`return`으로 빠져나가지 않은 경우).
- **표시할 선택지가 하나도 없으면**(모든 조건이 거짓) 런타임 에러가 납니다
  (`menu has no available choices`). 항상 조건 없는 "기본" 선택지를 하나 두는 것을 권장합니다.
- 선택지가 0개인 `menu:`(본문에 선택지 줄이 없음)는 **파싱 에러**입니다.

> 팁: 선택지 본문에서 `jump`로 라벨로 나가면 "완전 분기", `jump` 없이 변수만 바꾸면 "공통 흐름으로 합류"가 됩니다.

## 7. 조건 분기 — `if` / `elif` / `else`

```vns
if 호감도 >= 50 and 만난적:
    요르 "우리 사귈래?"
elif 호감도 >= 0:
    요르 "그냥 친구로 지내자."
else:
    요르 "우리 안 맞는 것 같아."
```

- `if <식>:` 로 시작하고, 이어서 `elif <식>:` 를 **0개 이상**, 마지막에 `else:` 를 **0개나 1개** 둘 수 있습니다.
- `elif`/`else`는 짝이 되는 `if`와 **같은 들여쓰기 깊이**여야 합니다(더 깊거나 얕으면 다른 블록으로 취급).
- 위에서부터 첫 번째로 참인 분기의 본문만 실행하고 나머지는 건너뜁니다. 모두 거짓이고 `else`가 있으면 `else` 본문을 실행합니다.
- 조건식 규칙은 [02번 문서](02-expression-language.md) 참고. `and`/`or`/`not`, 비교, 산술 모두 가능합니다.
- `elif`/`else`를 짝 없이 단독으로 쓰면 파싱 에러(`'elif' without matching 'if'`).

---

## 8. 반복 — `while`

조건이 참인 동안 본문을 반복합니다. 자원 틱·세대 루프 같은 시뮬레이션 계산에 씁니다.

```vns
while 남은턴 > 0:
    $ 골드 = 골드 + 채집량
    $ 남은턴 -= 1
```

- 문법: `while <식>:` + 들여쓴 본문.
- 조건을 매 반복 시작에 검사하고, 거짓이 되면 루프를 빠져나갑니다.
- **무한 루프 방지**: VM은 한 번의 `Tick()`에서 실행하는 명령 수에 상한(기본 100,000)이 있습니다.
  대사(`say`)나 선택지(`menu`)가 없는 루프가 이 상한을 넘으면 런타임 에러로 중단됩니다.
  → **본문에 대사가 없는 순수 계산 루프는 반드시 유한 회수로 끝나게** 작성하세요(위 예처럼 카운터 감소).

## 9. 변수 대입 — `$`

`$`로 시작하는 줄은 변수 대입입니다. 변수는 미리 선언할 필요 없이 처음 대입하면 생깁니다.

```vns
$ 골드 = 100
$ 호감도 += 30          # 호감도 = 호감도 + 30
$ 골드 -= 10
$ 점수 = (용기 + 지혜) * 2
$ 만난적 = true
```

- 단순 대입: `$ <변수> = <식>`
- 복합 대입: `+=` `-=` `*=` `/=` — `$ x += a` 는 `$ x = x + a`와 같습니다.
- 오른쪽에는 [표현식](02-expression-language.md)이 옵니다(변수·산술·`random()`·비교·논리 전부 가능).
- 정의 안 된 변수를 식에서 읽으면 `0`입니다. 그래서 `$ 골드 += 10`은 처음이라도 `0 + 10 = 10`이 됩니다.

## 10. 흐름 제어 — `label` · `jump` · `call` · `return`

### 라벨 `label`
```vns
label 데이트:
    ...
```
- `jump`/`call`의 목적지입니다. **정수 인덱스를 대체**하는 이름표입니다.
- 라벨 이름은 **모든 `.vns` 파일에 걸쳐 전역**입니다(파일 경계를 넘어 참조 가능).
- **같은 이름의 라벨을 두 번 정의하면 컴파일 에러**(`duplicate label`).

### 점프 `jump`
```vns
jump 데이트          # 복귀 없이 그 라벨로 이동
```
- 복귀 지점을 남기지 않고 이동합니다. 존재하지 않는 라벨이면 컴파일 에러(`unknown label`).

### 호출/복귀 `call` / `return`
```vns
call 공통_인트로     # 실행 후 이 줄 다음으로 돌아옴
...
label 공통_인트로:
    나레이션 "..."
    return           # call 지점 다음으로 복귀
```
- `call`은 복귀 주소를 콜스택에 쌓고 라벨로 점프합니다. 라벨 블록이 `return`을 만나면 **`call` 다음 명령으로 복귀**합니다.
- 콜스택이 비어 있는 상태(= 최상위 흐름)에서 `return`을 만나면 **스크립트가 종료**됩니다.
- 프로그램은 마지막에 자동으로 종료(Halt)됩니다. 즉 명시적 `return` 없이 코드 끝에 도달해도 정상 종료합니다.

## 11. 여러 파일 · 진입점

- `Assets/Resources/<폴더>/` 아래 모든 `.vns`가 **파일 이름(오디널) 순으로 정렬되어** 하나의 프로그램으로 합쳐집니다.
- `label`·`jump`·`call`은 파일 경계를 자유롭게 넘나듭니다(전역 네임스페이스).
- 실행 **진입 라벨**은 `VNRunner`의 `entryLabel` 필드로 지정하며 기본값은 `start`입니다.
  진입 라벨이 없으면 시작 시 런타임 에러(`entry label '...' not found`).

## 12. 흔한 에러와 원인

| 에러 메시지(발췌) | 원인 |
|---|---|
| `tab used in indentation` | 들여쓰기에 탭 사용 → 스페이스로 교체 |
| `'elif'/'else' without matching 'if'` | `elif`/`else`의 들여쓰기가 `if`와 안 맞거나 단독 사용 |
| `menu choice must start with a quoted label` | 선택지 줄이 `"`로 시작하지 않음 |
| `menu choice header must end with ':'` | 선택지 헤더 끝에 `:` 누락 |
| `menu has no choices` | `menu:` 아래 선택지 줄이 없음 |
| `menu has no available choices` (런타임) | 모든 선택지의 `if` 조건이 거짓 → 조건 없는 기본 선택지 추가 |
| `duplicate character` / `duplicate label` | 같은 id/라벨 중복 정의 |
| `unknown label '...'` | `jump`/`call` 대상 라벨이 없음(오타·파일 누락) |
| `character requires name:"..."` | `character`에 `name:` 누락 |
| `unknown position '...'` | `show`의 위치가 left/center/right가 아님 |
| `assignment needs an operator` | `$` 줄에 `=`/`+=` 등 대입 연산자 없음 |
| `step limit exceeded within one Tick` (런타임) | 대사 없는 `while` 루프가 안 끝남 → 종료 조건 확인 |

> 모든 파싱/컴파일 에러는 `파일이름:줄번호: 메시지` 형식으로 위치를 알려줍니다.

## 13. 전체 예제

```vns
# 요르 데이트 시나리오
character 요르 name:"요르" color:"#8fd3ff"
character 나레이션 name:"나레이션"

label start:
    bg city_sunset
    show 요르 left
    요르 "주말에 뭐 할래?"
    menu:
        "같이 영화 보자":
            $ 호감도 += 30
            jump after
        "선물을 준다 (골드 10)" if 골드 >= 10:
            $ 골드 -= 10
            $ 호감도 += 50
            jump after
        "귀찮은데 그냥 집에":
            $ 호감도 -= 10
            jump after

label after:
    나레이션 "...그렇게 시간이 흘렀다."
    if 호감도 >= 50:
        요르 "너랑 있으면 정말 즐거워. 우리 사귈래?"
    elif 호감도 >= 0:
        요르 "그냥 친구로 지내자."
    else:
        요르 "...우리 안 맞는 것 같아."
    return
```
