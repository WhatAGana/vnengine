# 스토리 ↔ 시스템 정합성 매핑 (v2.13 story_bible 기준)

> **목적**: story_bible(v2.13)의 서사 설정과 실제 구현된 코드(06~07-C) 사이의 대응을 명시.
> 클로드 코드가 1회차 뼈대·서사 씬을 만들 때, "이 서사가 어떤 코드를 부르는지" 참조용.
> **판정 기준일**: 07-C 완료 시점(EditMode 430/430). 코드 = main 머지분.
> **핵심 결론**: 스토리와 시스템은 정합. 스토리가 시스템을 예측하고 쓰여 충돌 거의 없음.
> 조정 필요 항목은 전부 나중 회차(6회차·온천·살의판정) 기능 → 1회차 뼈대엔 불필요.

## 2. 조정 필요 (전부 나중 회차 — 1회차 뼈대엔 불필요)

### A. 여관 온천 기능 — 미구현 (잎사귀)
- story: §8.2 온천(제국 부인 방문, 부패 연출, 제국에 유명해짐).
- code: 07-D 여관은 직원/메뉴/내구도까지. 온천 없음.
- 판정: **여관 타이쿤 확장 기능**. 1회차 필수 아님. 나중 여관 확장 슬라이스에서 InnState에 온천 필드 추가.
- 서사 연동: 온천은 회차 진행·제국 인지도와 물림 → 2회차+ 첩보와 연결. 1회차엔 없어도 됨.

### B. 여관 "살의 판정" 통합 로직 — 분리 구현 상태
- story: §8.1 코어가 살의를 읽어 손님/침입자를 한 규칙으로 구분.
- code: 현재 침입자(웨이브)와 손님(InnState)이 **별도 구현**. "살의 판정"이라는 통합 게이트는 없음.
- 판정: 1회차엔 불필요(모든 웨이브=침입자로 충분). DEFECTOR(생존욕구로 손님되는 병사, 부록A origin)가
  나오는 시점(나중 회차)에 통합 판정 로직 필요. 그때 CaptureRule 옆에 GuestRule 추가.

### C. 6회차 대칭 기억(둘 다 기억) — 코드에 개념 없음
- story: §5.2 v2.13 "6회차부터 주인공도 기억 유지=기본값" (1~5회차 비대칭 → 6회차~ 대칭).
- code: "주인공 기억" 개념 자체가 MetaState에 없음.
- 판정: **한참 뒤(6회차) 기능.** 1회차 뼈대엔 전혀 안 급함. 나중 서사 슬라이스에서 MetaState에
  "기억 플래그/모드" 추가. 지금 고려 불필요.

## 3. 1회차 뼈대에 실제로 필요한 것 (§6.1만)

> 스토리 전체(5회차+무한루프) 중 **1회차(§6.1)만** 추리면, 필요한 시스템은 전부 이미 코드에 있음.
> 새로 만들 시스템 없음. 있는 걸 90일 구조로 묶고 서사 씬만 얹으면 됨.

### 1회차 서사 비트 (§6.1)
1. 기상 → **악필 편지**("여관을 부탁한다", 미래-주인공이 쓴 것을 미갈이 보관하다 놓음) 발견
2. 영문 모르고 디펜스 시작
3. 미갈 등장 — 회귀를 아는 듯 의미심장한 말 흘림(학습된 무기력의 무방비한 누출, §7.7)
4. 튜토리얼 소환 — "무언가, 코어에서 이질감이 느껴진다"(§8.3 복선, [3편]=플레이어)
5. 9주차경 코어 자연 붕괴 → 영문 모르는 주인공 첫 회귀

### 각 비트 ↔ 코드
| 비트 | 필요 시스템 | 코드 상태 |
|---|---|---|
| 편지·미갈 대사·독백 | VN 씬 재생 | ✅ .vns 엔진(파서+VM+세이브) 있음 |
| 디펜스(90일) | 전투·배치·웨이브 | ✅ 07-A2/B, CampaignWaveRule.ResolveWave |
| 튜토리얼 소환 | 가챠(고정결과) | ✅ 07-C GachaRule (고정 시드로 튜토 결과 가능) |
| 여관 운영 | 여관 수급 | ✅ 07-D InnIncomeRule |
| 포획(로스터 씨앗) | 포획 | ✅ 07-B CaptureRule |
| 9주차 회귀 | 회귀 | ✅ 06 Regress |
| 90일 진행 | 시간구조 | ❗ 미구현 — 정비/웨이브 페이즈를 90일 루프로 묶는 것 (1회차 뼈대의 핵심 신규 작업) |

→ **1회차 뼈대의 유일한 신규 작업 = 90일 시간구조 + §6.1 서사 씬(.vns) + 둘 엮기.** 나머지는 배선만.

## 5. 서사↔시스템 연동 시 주의 (집필/구현 공통)

- **§8.3 "무언가의 의지" 복선**: 튜토리얼 소환의 고정 결과를 "시스템 강제"로 노출하지 말 것. 주인공 독백
  ("코어에서 이질감…")으로 연출. 코드는 고정 시드로 결과를 강제하되, **UI/텍스트는 그걸 "무언가의 의지"로 감쌈.**
  → 1편=미갈 추정, [3편]=플레이어. 미갈은 긍정/부정 안 함.
- **§5.2 이레귤러 기억**: 1회차는 주인공이 아무것도 기억 못 하는 상태(백지). 편지도 "영문 모름". 2회차부터
  편지를 정보로 받기 시작. → 1회차 서사 씬에서 주인공은 회귀·미갈 정체를 전혀 모르는 톤.
- **§7.9 미갈 말투 1회차 톤**: 학습된 무기력(어차피 잊을 사람이라 여긴 무방비). 의미심장하되 체념 섞인.
  → 대사 집필 시 story_bible §7.9 참조.
- **로스터 리셋 플래그**: 07-B에서 ResetPolicy를 열어만 뒀음. §9.1a가 "회차마다 리셋+이레귤러 해금"으로
  확정했으므로, 로스터 코드화 시 이 규칙으로 세팅(리셋되되 진행플래그는 메타 유지).

- **1회차 뼈대 신규 작업 = 90일 시간구조 + §6.1 서사 씬(.vns) + 엮기.** 새 게임 시스템 없음.

## 7. v2.14 신규 반영 (해방카드 · 2회차첩보 · 코스믹호러)

> story_bible v2.13→v2.14 변경 대조. 결론: **코드와 충돌 없음. 신규는 전부 2~3회차 콘텐츠라 1회차 뼈대엔 불필요.**

### 7.1 해방카드 시스템 (§9.1b) — v2.14 최대 신규, 미구현
- story: 서브히로인·첩보인물을 **배치유닛이 아니라 후방 버프 카드**로 구현. 슬롯 3개 + 카드 다수 → 조합 전술.
  캐릭터당 카드 2장(일반/진). 진 카드 = 서사 결착이 질적 능력으로.
- code: **미구현.** 07까지는 몹 배치(전선)만. "버프 카드 + 슬롯" 후방 시스템 없음.
- 판정: **신규 슬라이스 필요**(08급 새 시스템). 단 06 런/메타와 이미 정합 —
  story §9.1b가 명시: "카드 보유·진행도·등급=MetaState / 회차 내 슬롯 배치=RunState". → 만들 때 06 위에 얹힘.
- **1회차 무관**: 카드는 로스터 씬 진행·2회차 첩보에서 생성/활성. 1회차엔 로스터 씬1 정도만, 카드 활성화(2단계)는 후반 회차.
  → 1회차 뼈대에 카드 시스템 불필요. 1회차 돌린 뒤 2회차 만들 때 별도 슬라이스.

### 7.2 카드 활성화 2단계 (§9.1b·§5.6) — 시간차 로직, 나중
- story: ① 매듭 끊기(폭로/회개/씬진행) → 카드 생성(잠김) ② 인과 해소 사건(처형소식 등) → 카드 활성(슬롯 등록 가능).
- code: 미구현. 07-B 포획(포획→감옥→방면)과 결이 비슷하나 별도 시스템.
- 판정: 카드 슬라이스에서 구현. 1회차 무관.

### 7.3 2회차 첩보 SFW 재설계 (§6.2) — 서사 변경, 코드 무관
- 이전 미결(2회차 첩보=NSFW 프레임)이 v2.14에서 **7대 죄악 인과 프레임 + 단죄·인과 회수·카드화**로 재설계.
- 판정: 순수 서사 변경. 코드 영향 0. 2회차 콘텐츠라 1회차 무관.

### 7.4 코스믹호러화(초대 황제, §6.3) — 조건부 VN 대사 훅
- story: 초대 황제 = 억겁의 인과 코스믹호러. "주인공이 2회차 인과카드 세팅 시 추가 비웃음 대사"(방심 붕괴).
- code: 카드 보유상태(MetaState) 읽어 VN 대사 분기 → 06 MetaProjection 위에 얹힘. 단 카드 없어도 격은 전투로 전달(대사는 보너스).
- 판정: 3회차 콘텐츠 + 카드 시스템 선행 필요. 1회차 무관.

### 7.5 기존 정합 유지
- 회귀규칙·가챠·포획트리거·로스터트리거·런/메타 등 §1 매핑은 v2.14에서 불변.

### 7.6 v2.14 종합
- **충돌 없음.** 최대 신규(해방카드)가 06 런/메타를 명시 참조하며 설계 → 나중 구현 시 그 위에 얹힘.
